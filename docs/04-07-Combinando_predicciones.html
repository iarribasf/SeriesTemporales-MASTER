Combinación de predicciones

Previsión con Datos Temporales (GBIA)

Author

Iván Arribas (Depto. Análisis Económico. Universitat de València)



1 Antecedentes

En 2019 tuvo lugar el cuarto concurso M (M4C, en adelante), donde M viene de Makridakis. Los concursos M son una serie de concursos abiertos para evaluar y comparar la precisión de distintos métodos de previsión de series temporales. En concreto el M4C pedía diseñar un método de predicción y aplicarlo a 100000 series de todo tipo en cuanto a su naturaleza, longitud o estacionalidad. El método ganador, según una determinada medida de precisión indicada en las bases del concurso, ganaba 9000€. Este tipo de concursos es organizado por equipos dirigidos por el investigador Spyros Makridakis (más información aquí y aquí).

Un resumen de los resultados del M4C pueden verse en Makridakis et al. (Makridakis, Spiliotis, and Assimakopoulos 2020). En esta píldora, el aspecto a destacar es que de los 10 mejores métodos, 8 consistían en combinar las predicciones de diferentes metodologías (véase la tabla 4 del articulo referido). Es decir, se obtienen diferentes predicciones usando métodos ya conocidos, que luego se promedian dando lugar a la predicción final.

En la combinación de predicciones los elementos clave son: i) los métodos utilizados para obtener las diferentes predicciones, y ii) los pesos que se usan para combinarlas. Por ejemplo, en el M4C uno de los métodos usados como referencia consistía en obtener la media simple de las predicciones obtenidas con Alisado exponencial simple, Alisado de Holt y Alisado de Holt con amortiguamiento, métodos que has visto durante el curso. Este método combinado ocupaba la posición 19 en el ranking, mientras que los tres métodos individuales ocupaban las posiciones 33, 32 y 21, respectivamente.

En el M4C las posiciones 2 a 7 del ranking las ocuparon métodos consistentes en combinar predicciones. En concreto, el método de combinación que ocupó el segundo puesto del ranking fue diseñado por Montero-Manso et al. (2020). En el combinaban las predicciones de 9 métodos (Ingenuo con y sin estacionalidad, Deriva, Ets, Arima…) y los pesos se obtenían a partir de una función que utilizaba 42 características de la serie (longitud, medida de la fuerza de la tendencia y la estacionalidad, valores de la FAC, orden estacional…)

En esta píldora vamos a ver a través de un ejemplo como se pueden mejorar las predicciones de una serie combinando las predicciones obtenidas con diferentes métodos.


2 Metodología

Para el ejemplo usaremos como serie de referencia el Aforo de vehículos a su paso por la N-340 a la altura de Oropesa.

aforo <- read.csv2("./series/Aforo_oropesa.csv", 
                   header = TRUE)

aforo <- ts(aforo[, 1], 
            start = 1960, 
            freq = 1)

autoplot(aforo, 
         xlab = "", 
         ylab = "Vehículos diarios",
         main = "")

Vamos a considerar cuatro métodos de predicción vistos durante el curso:

  1. Ingenuo I (naive)
  2. Método de la deriva (rwf con drift = TRUE)
  3. Holt sin amortiguamiento (ets con model = "AAN y damped = FALSE)
  4. Holt con amortiguamiento (ets con model = "AAN y damped = TRUE)

Ajustaremos cada uno de estos cuatro métodos a la serie y obtendremos previsiones a 5 años vista. El resultado, por tanto, serán cuatro conjuntos de predicciones diferentes. Sea \((\hat{y}_1^n,\; \hat{y}_2^n,\; \hat{y}_3^n,\; \hat{y}_4^n,\; \hat{y}_5^n)\) las previsiones a cinco años vista obtenidas con el método \(n\), donde \(n = 1, \ldots, 4\).

Como indicador de calidad de las predicciones usaremos el MAPE.

Finalmente, para combinar las cuatro predicciones vamos a considerar dos medias ponderadas. En el primer caso daremos la misma importancia a todas las predicciones, es decir, la predicción combinada será la media simple de las cuatro predicciones. Por tanto, la previsión final para un horizonte de \(h\) años vista, \(h = 1,\ldots,5\), es

\[\hat{y}_h^S = \frac{1}{4}(\hat{y}_h^1 + \hat{y}_h^2 + \hat{y}_h^3 + \hat{y}_h^4).\]

En el segundo caso usaremos como pesos para el cálculo de la previsión media la inversa del RMSE de ajuste de cada método. Así, la predicción de un método con peor calidad de ajuste (mayor RMSE), tendrá un peso menor en la combinación. Si denominamos \(w_n = 1/RMSE_n\) al peso de la previsión obtenida con el método \(n\), entonces

\[\hat{y}_h^P = \frac{w_1 \hat{y}_h^1 + w_2 \hat{y}_h^2 + w_3 \hat{y}_h^3 + w_4 \hat{y}_h^4}{w_1 + w_2 + w_3 + w_4}.\]


3 Resultados

Veamos primero la calidad de ajustes obtenida con cada método

Podemos ver que la calidad de ajuste, medida a través del MAPE, oscila entre 7.25% para el Alisado de Holt sin amortiguamiento y el 8.3% para el método Ingenuo I. Por otro lado, usaremos las inversas de los RMSE de ajuste como pesos para una de las combinaciones de predicciones.

w1 <- 1 / accuracy(naive(aforo))[2]
w2 <- 1 / accuracy(rwf(aforo, drift = TRUE))[2]
w3 <- 1 / accuracy(ets(aforo, model = "AAN", damped = FALSE))[2]
w4 <- 1 / accuracy(ets(aforo, model = "AAN", damped = TRUE))[2]

Para obtener el indicador de precisión de las predicciones de cada uno de los cuatro métodos y de las dos combinaciones consideradas, usaremos origen de predicción móvil. Consideraremos que son necesarios 30 años para ajustar bien los modelos y haremos previsiones a 5 años vista.

k <- 30
h <- 5
TT <- length(aforo)
s <- TT - k - h

mapeIng <- mapeDer <- mapeHol <- mapeHod <- mapeCo1 <- mapeCo2 <- matrix(NA, s + 1, h)

for (i in 0:s) {
  train.set <- subset(aforo, start = i + 1, end = i + k)
  test.set <-  subset(aforo, start = i + k + 1, end = i + k + h)
  
  fcast <- naive(train.set, h = h)
  preIng <- fcast$mean
  mapeIng[i + 1,] <- 100 * abs(test.set - preIng)/test.set
  
  fcast <- rwf(train.set, h = h, drift = TRUE)
  preDer <- fcast$mean
  mapeDer[i + 1,] <- 100 * abs(test.set - preDer)/test.set

  fit <- ets(train.set, model = "AAN", damped = FALSE)
  fcast <- forecast(fit, h = h)
  preHol <- fcast$mean
  mapeHol[i + 1,] <- 100 * abs(test.set - preHol)/test.set

  fit <- ets(train.set, model = "AAN", damped = TRUE)
  fcast <- forecast(fit, h = h)
  preHod <- fcast$mean
  mapeHod[i + 1,] <- 100 * abs(test.set - preHod)/test.set
  
  preCo1 <- (preIng + preDer + preHol + preHod ) / 4
  mapeCo1[i + 1,] <- 100 * abs(test.set - preCo1)/test.set
  
  preCo2 <- (w1 * preIng + w2 * preDer + w3 * preHol + w4 * preHod) / 
            (w1 + w2 + w3 + w4)
  mapeCo2[i + 1,] <- 100 * abs(test.set - preCo2)/test.set
}

mapeIngMedia <- colMeans(mapeIng)
mapeDerMedia <- colMeans(mapeDer)
mapeHolMedia <- colMeans(mapeHol)
mapeHodMedia <- colMeans(mapeHod)
mapeCo1Media <- colMeans(mapeCo1)
mapeCo2Media <- colMeans(mapeCo2)
round(mapeIngMedia, 2)
[1]  4.36  9.53 13.56 17.19 20.23
round(mapeDerMedia, 2)
[1]  4.49 10.07 14.24 17.88 20.30
round(mapeHolMedia, 2)
[1]  5.22 12.00 18.50 24.96 28.83
round(mapeHodMedia, 2)
[1]  4.46 10.60 15.20 19.74 22.24
round(mapeCo1Media, 2)
[1]  4.18  9.44 13.47 17.33 19.75
round(mapeCo2Media, 2)
[1]  4.19  9.45 13.49 17.36 19.80

La predicción calculada como media simple de las predicciones de los cuatro métodos es la más precisa con independencia del horizonte temporal. Por ejemplo, a un periodo vista la precisión es del 4.18%, mientras que el mejor método (Ingenuo I) tiene una precisión del 4.36%; a cinco periodos vista la precisión es del 19.75%, mientras que el mejor método (de nuevo el Ingenuo I) tiene una precisión del 20.23%.1

La media ponderada ofrece predicciones prácticamente iguales que la media simple.
